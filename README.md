# DB-Backup-Utility
**Мульти-СУБД поддержка:** Единый интерфейс для работы с PostgreSQL и MySQL.  
**Полный цикл операций:** Создание бэкапов (backup) и восстановление данных (restore).  
**Управление конфигурацией:** Сохранение параметров подключения в защищенных профилях (через Viper).  
**Проверка соединения:** Валидация учетных данных перед выполнением операций.  
  
## Основные технологии:
**Cobra** Создание структурированного CLI с поддеркой подкоманд и флагов.  
**Viper** Управление конфигурацией (чтение YAML/JSON, работа с ENV переменными).  
**pgx** Высокопроизводительный драйвер для PostgreSQL.  
**go-sql-driver/mysql** Надежный драйвер для взаимодействия с MySQL.  
**minio** Работа с протоколом S3  
**Standard Library** Использование утилит (pg_dump, mysqldump) для гарантированной целостности данных.  
  
Для корректной работы утилиты в системе должны быть установлены клиентские утилиты соответствующих СУБД:  
**Для PostgreSQL:** pg_dump и psql  
**Для MySQL:** mysqldump и mysql  
  
Утилита использует файл config.json (src/config.json) для хранения профилей.  
  
## Команды  
`./dbtool --help` - справка по всем командам  
`./dbtool [command] --help` - справка по флагам для конкретной команды  
`./dbtool connect` - проверка корректности введённых данных и возможности подключения к бд  
`./dbtool backup` - создание резервной копии в src/files  
`./dbtool restore` - восстановление БД из резервной копии в src/files  
Файл для резервной копии в postgres имеет расширение `.dump`, в mysql - `.sql`  
  
**Значение флагов для работы с бд:**  
**database** - "postgres" для postgres, "mysql" для mysql  
**usecloud** - использование облачного хранилища или нет  
Процесс работы с облаком:  
**backup** - запись резервной копии в `src/files`, а после загрузка этого файла в облако  
**restore** - выгрузка из облака файла в `src/files`, а после использование в бд  
## Сборка  
```Bash
git clone https://github.com/Egor-Iv00/DB-Backup-Utility.git
cd DB-Backup-Utility/src
go build -o dbtool
```
Для более удобной работы с утилитой записывайте значения флагов в `src/config.json`
